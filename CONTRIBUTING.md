# Contributing

Linux Backup Manager is a personal project. It is shared publicly so that others can read,
use, and adapt it, not because it is seeking contributors or maintainers.

## What this means in practice

* **No support is provided.** The project is maintained on a best-effort, whenever-there-is-time
  basis for its own use case first.
* **Issues are disabled.** This is intentional, not an oversight — please do not ask for them
  to be re-enabled.
* **Pull requests may be opened**, since forking is a normal part of public GitHub repositories,
  but there is no commitment to review, respond to, or merge them. They may be left open
  indefinitely, closed without comment, or acted on eventually — entirely at the maintainer's
  discretion.
* **If you want to change something for yourself:** fork the repository. That is the intended
  way to adapt Linux Backup Manager to your own needs.

## Reporting a security vulnerability

Security issues are the one exception to "no support" — see `SECURITY.md` for the private
reporting process. Please do not open a public issue or pull request for a security problem.
